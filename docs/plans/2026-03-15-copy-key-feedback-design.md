# 密钥点击复制反馈设计

## 背景

当前控制台里的账号密钥和池密钥都支持点击复制，但复制逻辑分散在两个位置，并且会在未确认复制结果时直接显示成功状态。

## 目标

- 点击密钥后可以一键复制
- 复制成功后密钥文本变绿
- 左侧标签切换为“已复制！”
- 成功态约 1.5 秒后自动恢复
- 复制失败时不显示成功态

## 方案

采用统一的前端复制反馈逻辑，封装成可复用组件或 Hook，让账号密钥和池密钥共享同一套行为。

### 交互

- 默认显示“API 密钥：”或“池密钥：”
- 点击密钥文本时触发复制
- 复制成功后将文本颜色切换为 `var(--green)`，同时标签切换为“已复制！”
- 1.5 秒后清除成功态

### 错误处理

- 复制依赖 `clipboard` 工具模块
- 使用 Promise 返回复制是否成功
- 失败时不进入成功态，避免误导用户

### 范围

- `web/src/components/AccountCard.tsx`
- `web/src/App.tsx`
- `web/src/clipboard.ts`
- 前端测试与测试配置文件
